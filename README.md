# Google Takeout Metadata Embedder

A Python utility for embedding `photoTakenTime` metadata from Google Takeout JSON sidecar files directly into the EXIF and metadata headers of your photos and videos.

## Overview

When downloading assets from Google Photos via Google Takeout, metadata such as the original creation date is segregated into supplementary `.json` files. This script processes your Takeout folder, pairs each media file with its corresponding JSON sidecar—even accounting for varied truncation and duplication naming patterns and embeds the timestamp back into the media file using ExifTool.


## Prerequisites

1. **Python 3.7+**
2. **ExifTool:** Ensure [ExifTool](https://exiftool.org/) is installed and added to your system's `PATH`.

## Usage

Run the script from your terminal, passing the path to the Takeout folder as an argument.

```bash
python embed_metadata.py "PATH_TO_TAKEOUT_FOLDER"
```

### Options

- `folder`
  The main directory generated by Google Takeout. All subdirectories will be recursively scanned.
  
- `--dry-run`
  Logs the intended actions without mutating the files. Recommended for verification prior to execution.
  
- `--process-trashed`
  Includes files prefixed with `.trashed-`, which otherwise represents media deleted in Google Photos.

### Example

Test the process on your extracted Takeout export without mutating files:
```bash
python embed_metadata.py "/Users/Saqin/Downloads/Takeout" --dry-run
```

Apply the metadata officially:
```bash
python embed_metadata.py "D:\Takeout"
```

## How It Works

1. **Recursive Scan**: Identifies all valid image (`.jpg`, `.png`, `.heic`, etc.) and video (`.mp4`, `.mov`, etc.) formats.
2. **Metadata Locating**: Analyzes the root filename and evaluates it against multiple known Google generated sidecar suffixes and pattern variations.
3. **Timestamp Application**: Parses the best available creation timestamp (`photoTakenTime` preferentially) and commands `exiftool` to overwrite the corresponding EXIF date properties (e.g. `DateTimeOriginal`, `CreateDate`, `ModifyDate`).
4. **Error Management**: Records any unmodifiable file names, absent metadata pairs, or corrupt files into individual text logs within the working directory.
